#卒研のAI提案

１. 建築AI（被験者役のBot）
使っているツール： Mineflayer ＋ Baritone ＋ 設計図（Litematic）

【① Mineflayer（五感）】
「設計図のこの座標に、まだブロックが置かれていないぞ！」とデータで検知 │ ▼（裏のパケット通信 / チャットは使わない） 
【② 建築AIのロジック（脳）】※設計図データ（Litematic）を持っている 
「よし、手持ちの素材を確認して、足りなければ集め、あるなら設置しよう」と判断 │
 ▼（裏からAPIで命令をパス） 
【③ Baritone（手足）】 
命令を受けて、移動・採掘（素材集め）・ブロック設置を実行
2. サポートAI（研究対象のBot）
使っているツール： Mineflayer ＋ Baritone ＋ OpenAI API　orルールを自分で書く

【① Mineflayer（五感）】 
「隣の建築AIが、木材を連続で設置した！」と検知 │ 
▼（裏のパケット通信 / チャットは使わない） 
【② サポートAIのロジック（脳）】※OpenAI APIに裏で相談 
「あいつは今、木造建築をしていて素材が足りないな。素材を取りに行った方がいい」と判断 │
 ▼（裏からAPIで命令をパス） 
OR
【② Mineflayer（脳）】※ルールのもと決められた行動をしたら決まった行動をする
「木材を使った→素材があるか確認→なかったら取りに行く」
 ▼（裏から命令をパス） 

【③ Baritone（手足）】 命令を受けて、近くの森へ走り、木を伐って戻ってきて足元に素材をそっと落とす


Mineflayer
あらかじめ「プレイヤーがこういう動きをしたら、この処理をする」というコードで細かく書いておくので、「何かを検知して動き始める（きっかけを作る）」のは大得意。
「あそこの崖を登って、素材を集めて、設計図のこの位置にハーフブロックを逆向きに設置して」といった複雑な命令をMineflayerだけでやろうとすると、何千行もの泥臭いコードを自分で書く羽目になり、バグだらけになって破綻する。

Baritone
マイクラ内の移動・建築・素材集めの能力は強いですが、自律的な意志はありません。外部から「これをやれ」という具体的な命令（タスク）が与えられないと、その場で一歩も動けず突っ立っているだけになります。
